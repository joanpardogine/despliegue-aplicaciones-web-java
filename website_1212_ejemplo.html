1.2.1.2.- Ejemplo.

A continuación, podemos ver un ejemplo de servlet:

import java.io.*;
import javax.servlet.*;
import javax.servlet.http.*;

public class Saludo extends HttpServlet {
   
   protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
      response.setContentType("text/html");
      PrintWriter out = response.getWriter();
      out.println("<html>");
      out.println("<head><title>Hola mundo</title></head>");
      out.println("<body>");
      out.println("<h1>Hola Mundo</h1>");
      out.println("<p>Request URI " + request.getRequestURI() + "</p>");
      out.println("<p>Protocolo: " + request.getProtocol() + "</p>");
      out.println("<p>Direccion remota: " + request.getRemoteAddr() + "</p>");
      out.println("<p>Numero aleatorio: <strong>" + Math.random() + "</strong></p>");
      out.println("</body></html>");
      out.close();
   }

    protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        doGet(request, response);
    }
}

La primera línea nos indica la forma en que se generará la pagina de respuesta, en este caso "texto HTML".

La segunda nos indica el objeto que obtiene de "PrintWriter" que se asocia a la salida de texto del Servlet, esto será la respuesta. Quiere decir, que todo lo que ingrese en "out.println()" será la respuesta que recibirá el cliente. Fíjate también, que hemos escrito código HTML entre comillas, dentro de código Java, que significa que estamos devolviendo una respuesta al cliente pero con un lenguaje que pueda leer el mismo cliente que es HTML.

Los diferentes métodos del objeto request devuelven diferentes valores obtenidos de la petición hecha por el cliente.

Cuando creamos un servlet, debemos definir los métodos doGet o doPost, dependiendo del tipo de peticiones que va a repetir. Si cada tipo de solicitud se va a procesar de manera diferente, tendremos que definir ambas.  Si las 2 van a hacerlo de la misma manera, podemos definir uno de los métodos y llamarlo desde el otro como se hace en el ejemplo. Todas las peticiones POST se procesan llamando al método doGet. Es decir, como si fueran consultas GET.
