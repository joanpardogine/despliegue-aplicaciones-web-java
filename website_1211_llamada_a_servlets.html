1.2.1.1.- Llamada a servlets.

Para poder ejecutar un servlet, como ocurre con cualquier otra página web, debemos indicar su URL y para ello, tenemos 2 alternativas:

  • Identificar esa URL en el fichero descriptor de despliegue
  • Identificarla en el propio servlet mediante las anotaciones de la clase del servlet

Si añadimos las siguientes líneas de configuración en el fichero descriptor de depliegue WEB-INF/web.xml, indicaremos que al servlet le vamos a llamar con la URL definida:

<servlet>
    <servlet-name>NombreServlet</servlet-name>
    <servlet-class>paquete1.subpaquete1.MiServlet</servlet-class>
</servlet>

<servlet-mapping>
    <servlet-name>NombreServlet</servlet-name>
    <url-pattern>/ejemploservlet</url-pattern>
</servlet-mapping>

Lo que hacemos es asignar a la clase del servlet un nombre (en este caso NombreServlet) y luego mapear ese nombre con una URL (en este caso /ejemploservlet). Así, podemos llamar al servlet con:

http://localhost:8080/miapp/ejemploservlet

A partir de la especificación 3.0 de los servlet, también podemos describir los servlets utilizando anotaciones dentro de la propia clase del servlet.

Para poder utilizar las anotaciones debemos de importar el paquete annotations y utilizar la anotación @WebServlet(“URL-Pattern”). Vamos a ver nuestros tres ejemplos utilizando las anotaciones:

import javax.servlet.annotations.*;

@WebServlet("/ejemploservlet")
public class MiServlet extends HttpServlet {
   ...
}

Fijate en la barra que escribimos antes de la URL. Es imprescindible y si la olvidamos, la ejecución del servlet nos dará error.

Además, si usamos los 2 métodos y definimos 2 URLs distintas, funcionarán ambas.
